## 背景・課題

### 企業のAI導入が失敗する3つの理由

#### 課題①：AI契約がバラバラで管理できない

| 問題 | 詳細 |
|------|------|
| **症状** | 社員が個人でChatGPT、Claude、Geminiに課金している |
| **原因** | ChatGPT派、Claude派、Gemini派がそれぞれ存在 |
| **結果** | 会社としてAI利用の実態が把握できない。セキュリティリスクも増大 |

#### 課題②：社内情報をAIに入れるハードルが高い

| 問題 | 詳細 |
|------|------|
| **症状** | ChatGPTに社内資料を入れたいが、セキュリティが心配 |
| **原因** | ChatGPT Team/Enterpriseは高額。無料版は学習に使われるリスク |
| **結果** | 結局、社内情報を活用できず、AIの価値が半減 |

#### 課題③：言われたことしかやらない

| 問題 | 詳細 |
|------|------|
| **症状** | 「提案書作って」と言えば提案書だけ作る |
| **原因** | 指示の背景にある「本当の目的」を理解しようとしない |
| **結果** | 作業は代行してくれるが、目的達成には至らない |

### 結果

> **70%のAIツールが導入後1年以内に利用停止**

## 法人AIの特徴

### 従来のAIとの違い

| | 従来のAI | 法人AI |
|---|----------|--------|
| **使えるAI** | 1種類のみ | **GPT・Claude・Gemini全部** |
| **社内情報** | 別契約が必要 | **最初から統合** |
| **会話の起点** | 「〇〇作って」 | **「何を達成したい？」** |
| **対応範囲** | 言われたことだけ | **必要なこと全部** |
| **提案** | しない | **「こうしましょうか？」** |
| **学習** | しない | **使うほど賢くなる** |

### 体験イメージ

**【従来のAI】**
```
ユーザー：「提案書作って」
AI：「はい、提案書を作成しました。」
ユーザー：「競合比較も入れて」
AI：「競合比較を追加しました。」
ユーザー：「想定Q&Aも作って」
AI：「想定Q&Aを作成しました。」
ユーザー：「Googleドライブに保存して... あ、できないのか」

→ 結局、全体設計と段取りは人間がやっている
```

**【法人AI】**
```
ユーザー：「提案書作って」
法人AI：「何のために必要ですか？」
ユーザー：「A社との契約を取りたい」

法人AI：「契約獲得に必要なこと全部やりますね：
        ① A社の課題分析
        ② 競合との差別化整理
        ③ 提案書本体
        ④ 想定Q&A
        ⑤ フォローメールのテンプレート

        全部準備しますか？」

ユーザー：「OK」

法人AI：「完了しました。
        ・全資料をGoogleドライブに保存済み
        ・プレゼン前日にリマインド設定
        ・先方からの返信があればSlackに通知します」
```

**「OK」と言っただけで、全部終わっている。**

## なぜアドネスがやるのか

### 三位一体の強み

| 要素 | アドネスの強み | 法人AIとの関係 |
|------|----------------|----------------|
| **研修** | AIの使い方を教えられる | 法人AIの活用スキル習得、定着率向上 |
| **コンサル** | 目的を言語化できる | 法人AIへの要件定義、目的の明確化 |
| **プロダクト** | それを実行するAIを作れる | 研修・コンサルの知見を製品に反映 |

**この3つが揃っている会社は、ほぼない。**

### 顧客獲得の導線

```
研修顧客 ──▶ コンサル顧客 ──▶ 法人AI顧客

・研修でAIの可能性を理解
　　　↓
・コンサルで自社の課題を言語化
　　　↓
・法人AIで解決を実行

→ 一気通貫で顧客の成功を支援
```
