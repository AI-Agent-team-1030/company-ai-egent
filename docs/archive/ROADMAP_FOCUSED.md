# 法人AI 集中ロードマップ（1ヶ月/3ヶ月）

**最終更新**: 2025年11月23日
**戦略**: 既存機能を磨き上げ、コア機能で勝負する

---

## 🎯 基本方針

### ✅ やること
- **既存機能の改良**を最優先
- コア機能を完璧に仕上げる
- 売上につながる機能に集中

### ❌ やらないこと
- 新機能の大量追加
- 「あったら便利」レベルの機能
- 使われない機能の保守

---

## 📊 機能の分類

### 🔥 コア機能（絶対に必要、ここで勝負）

#### 1. ホーム（旧：ダッシュボード）
```
【現状】
- リアルタイム進行状況
- タスク完了率、稼働中のAI数など

【改良後】
- 「組織の健康状態」を表示
  - 各部署の稼働状況
  - AIエージェントの活動状況
  - 組織全体のアクティビティ
- シンプルでわかりやすい
- 経営者が「会社の状態」を一目で把握できる
```

**優先度**: ⭐⭐⭐⭐⭐（最高）
**1ヶ月目**: 組織健康状態の表示を実装
**3ヶ月目**: リアルタイム更新、アラート機能

---

#### 2. AIチャット
```
【現状】
- 経営AIとの対話
- AIエージェント自動実行
- RAGで情報取得

【改良後】
- RAG最適化（全部じゃなく最適なものだけ）
- ストリーミング応答の改善
- 参照したナレッジ/ドキュメントを明示
- チャット履歴の改善
- 「ナレッジ化」ボタン（よく聞かれる質問を保存）
```

**優先度**: ⭐⭐⭐⭐⭐（最高）
**1ヶ月目**: RAG最適化、ストリーミング安定化
**3ヶ月目**: マルチエージェント連携の完成

---

#### 3. ナレッジベース
```
【現状】
- 組織の知識を蓄積・共有
- カテゴリ別の分類

【改良後】
- チャットから簡単にナレッジ化
- 「これ毎回聞かれるわ」→ ボタン1つで保存
- 部署内共有 or 全社共有の選択
- 使用回数・評価のトラッキング
- 検索の高速化
```

**優先度**: ⭐⭐⭐⭐⭐（最高）
**1ヶ月目**: チャットからのナレッジ化機能
**3ヶ月目**: 自動カテゴリ分類、レコメンド

---

#### 4. ドキュメント管理
```
【現状】
- レポート、提案書、マニュアル、契約書の管理
- タイプ別フィルタリング

【改良後】
- ファイルアップロード（PDF/DOCX）
- AIがテキスト抽出 + 検索可能に
- チャットで「〇〇の会議資料ある？」と聞ける
- 自動生成した成果物を自動保存
- 部署ごとのアクセス制御
```

**優先度**: ⭐⭐⭐⭐⭐（最高）
**1ヶ月目**: ファイルアップロード、テキスト抽出
**3ヶ月目**: 高度な検索、自動分類

---

#### 5. 組織管理
```
【現状】
- 組織図・マインドマップ表示
- 経営AI → 部門AI → 統合AIの3層構造

【改良後】
- 法人 → 部署 → ユーザーの階層管理
- 権限管理（部署ごとのデータ分離）
- ユーザー招待機能
- 組織図の可視化（誰がどの部署にいるか）
```

**優先度**: ⭐⭐⭐⭐（高）
**1ヶ月目**: 基本的な組織管理（法人・部署・ユーザー）
**3ヶ月目**: 詳細な権限管理、組織図の可視化

```
理由:
- ホームに統合（組織の健康状態として表示）
- 別ページにする必要なし
```

**対応**: analyticsページを削除 → ホームに統合

---

#### 削除: chat-simple
```
理由:
- chatと重複
- シンプル版は不要
```

**対応**: chat-simpleを削除 → chatに一本化

---

#### 削除: simple-agent
```
理由:
- agentsページに一本化
- 開発用のプロトタイプとして不要
```

**対応**: simple-agentを削除 → agentsに統合

## 🚧 リスクと対策

### リスク1: 1ヶ月で間に合わない
**対策:**
- スコープを絞る（ナレッジorドキュメント、どちらか1つに集中）
- 外部リソースの活用

### リスク2: 初期顧客が獲得できない
**対策:**
- デモ先を20社に増やす
- 無料トライアル期間を30日に延長
- 初期割引（50% off for 3 months）

### リスク3: RAG最適化が難しい
**対策:**
- Supabaseのハイブリッド検索を活用
- リランキングは簡易版から始める
- 必要ならLangChainやLlamaIndexを部分的に導入
